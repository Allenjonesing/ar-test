ARCore Geospatial
Outdoor Anchors

Native · Android · ARCore Geospatial API

The ARCore Geospatial API places WGS-84 anchors at real-world coordinates with 10–30 cm accuracy in areas covered by Google Street View imagery. Anchors are persisted to the backend and survive app restarts — enabling cross-session, shared geo-anchored content without cloud anchor infrastructure.

◈ Prerequisites

  • Android device with ARCore support
  • Google Play Services for AR installed
  • Google Cloud API key (ARCore API enabled)
  • Outdoors in Google streetscape coverage area
  • Good sky visibility (GNSS)
  • Native Unity build required

◈ Limitations

  • Android-only — ARCore Geospatial not available on iOS
  • Accuracy degrades outside Google streetscape coverage
  • Requires Google API key — paid usage beyond free tier
  • Requires native build — cannot run in browser
  • Complex setup compared to GPS-only experiments

◈ Best Use Case

Outdoor AR with persistent geo-anchored content in covered areas. Compare accuracy directly against Experiment B (GPS-only browser) to quantify the improvement from centimetre-scale geospatial anchoring.

  │     └── Creates ARGeospatialAnchor at lat/lng/alt/heading

◈ How to Build & Run
  1. Open native-unity-lab/ as a Unity 2022 LTS (or later) project.
  2. Install AR Foundation 5.x and ARCore XR Plugin with Geospatial feature enabled via Package Manager.
  3. Add your Google Cloud API key (ARCore API) to AndroidManifest.xml and the ARCore settings asset.
  4. Ensure android.hardware.camera.ar is listed in AndroidManifest.xml <uses-feature>.
  5. Open the Loader scene (Assets/Scenes/Loader.unity) as build entry point.
  6. Build an Android APK and deploy to an ARCore-supported device.
  7. Test outdoors in an area with Google Street View coverage for full precision.
