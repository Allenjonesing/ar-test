GIS / Engineering
Coordinate AR

Native · Android / iOS · GIS · AEC

AR anchors placed using a local engineering coordinate grid rather than WGS-84 GPS. Survey control points define a known local origin; GISCoordinateManager computes the transform from GIS grid to AR local space and projects all anchors accordingly. Designed for AEC/construction sites where millimetre-scale accuracy is required and RTK GPS provides control-point positions.

◈ Prerequisites

  • Survey control points in a known local grid
  • RTK GPS receiver (for mm–cm accuracy)
  • AR Foundation 5.x on Android or iOS device
  • Control point data loaded into the backend
  • Native Unity build required

◈ Limitations

  • Requires survey control points — not plug-and-play
  • Complex coordinate transformation setup
  • Accuracy bounded by control point survey quality
  • Requires native build — cannot run in browser
  • Not suitable for dynamic or unmapped sites

◈ Best Use Case

AEC / engineering AR overlay on surveyed construction or infrastructure sites. Utility mapping, as-built verification, structural inspection, and any workflow where survey-grade positional accuracy is required.

◈ Coordinate Transform Pipeline
GIS Grid Coords → Survey control points define local grid origin and orientation (e.g., site Easting/Northing)
Control Point Fit → GISCoordinateManager computes a rigid transform (translation + rotation + optional scale) from GIS grid to AR local space using ≥ 3 control points
AR World Space → All GIS anchors projected into Unity AR coordinate frame — AnchorController places objects at computed positions
Residual Display → DebugHUD shows coordinate residuals per control point — user can assess transform quality before trusting placed content
◈ Architecture — native-unity-lab
ExperimentF_GIS scene
  ├── GISCoordinateManager.cs
  │     ├── GET /scenes/:sceneId  (fetches survey control points from backend)
  │     ├── Computes rigid transform: GIS grid → AR local space
  │     └── Projects all GIS anchors into AR world space
  ├── AnchorController.cs        → places AR objects at projected positions
  └── DebugHUD.cs                → shows coordinate residuals per control point
Coordinate residuals: The DebugHUD shows the residual error at each control point after the best-fit transform is applied. Residuals > 5 cm indicate poor control point quality or insufficient point distribution — honest feedback, not silent failure.
◈ How to Build & Run
  1. Collect survey control points using an RTK GPS receiver at the target site. Record GIS grid (Easting/Northing/Elevation) and WGS-84 coordinates for each point.
  2. Add a SceneDefinition to the backend (via POST /scenes or directly in the database) with the control point set.
  3. Open native-unity-lab/ as a Unity 2022 LTS (or later) project with AR Foundation 5.x installed.
  4. Open the Loader scene (Assets/Scenes/Loader.unity) as the build entry point.
  5. Build for Android or iOS and deploy to an ARCore/ARKit device.
  6. On site, open the app and walk to each control point marker; the app will align the GIS grid with AR local space and display residuals.
