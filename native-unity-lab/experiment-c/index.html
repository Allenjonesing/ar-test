Unity AR Foundation
Local / Hybrid Tracking

Native · Android / iOS · AR Foundation

ARCore (Android) and ARKit (iOS) local world tracking with optional GPS overlay. Provides centimetre-scale local anchors — the baseline for native tracking quality comparison across all AR Lab experiments. Tap to place anchors; a drift warning appears after 5 minutes of continuous session.

◈ Device Requirements

  • Android device with ARCore support
  • — OR — iOS device with ARKit support
  • Google Play Services for AR (Android)
  • iOS 12+ (iOS)
  • Native build — not available in browser
  • HTTPS not required (native app)

◈ Limitations

  • Anchors do not persist across app restarts (no cloud anchors)
  • GPS overlay accuracy limited by consumer hardware (±5–15 m)
  • Local anchors and GPS overlay are not precisely aligned
  • Requires native build — cannot run in browser

◈ Best Use Case

Indoor AR prototyping; local anchor quality benchmarking; comparison baseline for Experiment D (geospatial) and Experiment E (VPS).

◈ Architecture — native-unity-lab
Loader Scene
  └── ExperimentLoader.cs
        ├── GET /experiments/experiment-c  (fetch config)

ExperimentC_LocalTracking scene
  ├── AR Session Origin
  │     └── AR Camera (ARCameraManager, ARCameraBackground)

  ├── AnchorController.cs      → manages ARAnchor lifecycle

◈ Tracking State Machine
INITIALIZINGWaiting for ARSession.state == Tracking
↓
TRACKINGPlane detected — tap to place anchor
↓
ANCHOR_PLACEDOne or more anchors placed in local space
↓
DRIFT_WARNINGAfter 5 min — banner shown to user
↓
ENDEDSession ends → POST /telemetry with session summary
GPS Overlay (optional): When configOptions.enableGpsOverlay is true, LocalTrackingManager also starts LocationService and projects GPS coordinates onto the local AR coordinate frame using the initial fix as the local origin.

Accuracy note shown to user: "GPS overlay ±5–15 m. Local AR anchors are cm-scale. The two do not match precisely."
◈ How to Build & Run
  1. Open native-unity-lab/ as a Unity 2022 LTS (or later) project.
  2. Install AR Foundation 5.x, ARCore XR Plugin, and ARKit XR Plugin via Package Manager.
  3. Open the Loader scene (Assets/Scenes/Loader.unity) as the build entry point.
  4. Configure Android Build Support (or iOS Build Support) module in Unity Hub.
  5. Build & Run to an ARCore-supported Android device or iOS ARKit device.
  6. The app fetches the experiment config from GET /experiments/experiment-c on launch; falls back to StreamingAssets/default-config.json if offline.
