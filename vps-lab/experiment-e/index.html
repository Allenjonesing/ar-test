Niantic Spatial VPS
Visual Positioning System

Native · Android / iOS · Niantic Lightship

Niantic's Visual Positioning System (VPS) localizes a device to a sub-metre pose at a pre-scanned wayspot using visual feature matching against a stored 3D map of that location. Accuracy is typically < 1 m — unlike GPS, VPS provides a repeatable, drift-resistant pose at each visit.

◈ Performance Metrics

Accuracy< 1 m at scanned wayspots
Drift riskNone (VPS re-localizes)
Anchor persistenceBackend (cross-session)
Localization timeout30 s before fail state
PlatformAndroid + iOS
Max anchors10 per wayspot

◈ Prerequisites

  • Niantic Lightship ARDK installed via UPM
  • Niantic developer account + API key
  • AR Foundation 5.x
  • Pre-scanned Niantic wayspot for testing
  • Physical Android or iOS device (VPS cannot be tested in editor)
  • Native Unity build required

◈ What VPS Is NOT

  • Not a general-purpose positioning system
  • Not available at arbitrary locations — only scanned wayspots
  • Not a substitute for GPS in outdoor navigation
  • Not usable without a Niantic developer API key

◈ Best Use Case

Persistent, shareable AR experiences at specific real-world landmarks. Best for museum installations, public art, retail activations, or any experience that returns users to the same physical location.

◈ Localization State Flow
App LaunchWayspotLoader.cs fetches wayspot list from backend — falls back to default-wayspots.json if offline
Wayspot SelectedUser selects wayspot from list; POST /sessions to obtain session token
ATTEMPTINGLocalizationMonitor polls IARLocalizationPayload — UI shows "Scanning… point device at the location"
LOCALIZED✓ Sub-metre accuracy confirmed — VPS-anchored content revealed — POST /calibration-events (vps-localised)
FAILED✗ Timeout after 30 s — offer Try Again / Exit — POST /calibration-events (vps-localisation-failed)
LOST⚠ Tracking lost after initial fix — triggers re-scanning — POST /calibration-events (relocalisation-triggered)

Content authoring: VPS-anchored content is stored on the backend, not hard-coded in the app. After localisation, VPSAnchorController calls GET /scenes/:sceneId and creates anchors at the poses returned. New content can be added at a wayspot via POST /anchors without a new app build.
◈ How to Build & Run
  1. Create a Niantic developer account at lightship.dev and obtain an API key.
  2. Open vps-lab/ as a Unity 2022 LTS (or later) project.
  3. Install Niantic Lightship ARDK and AR Foundation 5.x via Package Manager.
  4. Add your Niantic API key to the Lightship settings asset in the Unity project.
  5. Open the Loader scene (Assets/Scenes/Loader.unity) as the build entry point.
  6. Build for Android or iOS and deploy to a physical device.
  7. Test at a pre-scanned Niantic wayspot — use the Niantic Wayfarer app to scan new locations.

◈ Adding a New Location

  1. Scan the real-world location with the Niantic Wayfarer app to create a wayspot.
  2. Add a LocationDefinition to the backend with the Niantic vpsMapId.
  3. Place anchors via POST /anchors — no app rebuild required.
